Cron Health

      

        The aggregation cron is a scheduled task inside PocketBase that runs every hour
        on the top of the hour. It drains the merchant / paycheck / bill submission queues into
        the consensus tables, weights submissions by confidence, applies user-merge alias
        promotions, and copies vendor phone/domain onto canonical entities. If the cron
        stops firing, crowd-data accumulates in the queue but never aggregates — the
        seed library stops growing, alias promotions stall, and the flywheel stalls.
        


        This card reads a heartbeat that the hook writes on every run. Green
        if the last run was within 90 minutes (cron is healthy). Yellow
        90-180 min (might just be one missed cycle). Red if >180 min
        (cron is broken — investigate). Gray means the heartbeat
        collection doesn't exist yet — create _cron_heartbeat in PocketBase admin
        with fields name (text) and last_run (date), and the
        aggregation hook will start writing to it on the next run.
      

      

    


    
    

      
Quarantine Watch

      

        When the aggregation hook scans an incoming submission and detects something that
        looks like personally identifiable information (SSN, credit card, email, phone, routing
        number, account number), it routes the submission to _quarantine_submissions
        instead of feeding it into the consensus tables. Privacy-by-default:
        we'd rather false-positive a usable data point than leak PII into a public-read
        collection. Green at 0 records.
        Yellow at 1–10 records (review when convenient).
        Red at >10 (something's misclassifying — investigate the
        hook's PII patterns and check whether legitimate user data is getting trapped).
      

      

    


    
    

      
Recent Commits

      

        Last commits across website + app-source repos via GitHub's API. The website
        repo (josspatech.github.io) is public; commits are read without auth.
        App-source repos (PBJ / HHH / CVC) are private; commits require a fine-grained
        personal access token. To include the private repos:
        generate a PAT at
        github.com/settings/personal-access-tokens
        with read-only Contents permission on the three repos, then in the browser
        console (F12 → Console) run localStorage.setItem("gh_pat", "github_pat_…")
        and reload the page. The token stays per-browser — never committed to source.
        Until then, only website commits show; private repos are skipped silently.

What this dashboard is

        

          This is the JosspaTech Flywheel Ops dashboard. It watches the shared
          PocketBase instance that all three JosspaTech apps write to — PocketBudJet (PBJ),
          Handy Horology Helper (HHH), and Curator's Vault (CVC) — so you have one place
          to see whether the consensus engine is healthy instead of bouncing between PocketBase
          admin, GitHub Insights, PocketHost billing, and Google Play Console.
        

        

          The flywheel: users scan receipts / pay stubs / bills. PBJ writes anonymized
          submissions to PocketBase. Server-side hooks (deployed from this dashboard's deploy scripts)
          drain those submissions into canonical merchants/employers/payees. Every device then reads
          back richer intel on the next scan. Each user correction makes the database a little smarter
          for every future user.
        

        

          The cards above are the instrumentation. The buttons in Maintenance & Reports are
          the controls. If something's off, start with Week-over-week banner (top), then the
          red/yellow cards, then the anomaly & crash lists.

If something breaks

        
    
          
  • Dashboard shows "Last aggregation: never" — hooks aren't deployed. Run
          deploy_everything.ps1.
    • 
          
  • Queue card turns red (>500) — aggregation isn't keeping up. Click
          Run aggregation now in the Force-run section.
    • 
          
  • Force-run buttons return 404 — ops_routes.pb.js didn't
          load. Re-run deploy, then click Ping /api/ops/health to verify.
    • 
          
  • Crashes spike after a release — click into the Recent crashes
          list and group by error_name. Roll back the release if it's systemic.
    • 
          
  • Dashboard fetches get rate-limited — PocketHost added /api
          rate limits. Bump the refresh interval dropdown up top to 5m or off.
