What this dashboard is

        

          This is the Flywheel Dashboard for the PocketBudJet crowd-data system.
          It watches the PocketBase instance that PBJ, HHH, and CVC all share, so you have one
          place to see whether the consensus engine is healthy instead of bouncing between PocketBase
          admin, GitHub Insights, PocketHost billing, and Google Play Console.
        

        

          The flywheel: users scan receipts / pay stubs / bills. PBJ writes anonymized
          submissions to PocketBase. Server-side hooks (deployed from this dashboard's deploy scripts)
          drain those submissions into canonical merchants/employers/payees. Every device then reads
          back richer intel on the next scan. Each user correction makes the database a little smarter
          for every future user.
        

        

          The cards above are the instrumentation. The buttons in Maintenance & Reports are
          the controls. If something's off, start with Week-over-week banner (top), then the
          red/yellow cards, then the anomaly & crash lists.
        


        
Deploying updates

        

          There are four things that can change and how to ship each:
        


        
1. Server-side hooks (anything in pb_hooks/*.pb.js — aggregation
        logic, anomaly thresholds, rate limits, force-run routes).

        
cd C:\Users\jossp\Documents\MobileApps\ClaudeFiles
powershell -ExecutionPolicy Bypass -File deploy_everything.ps1

GitHub Pages rebuilds in 30–60 seconds.

If something breaks

        
    
          
  • Dashboard shows "Last aggregation: never" — hooks aren't deployed. Run
          deploy_everything.ps1.
    • 
          
  • Queue card turns red (>500) — aggregation isn't keeping up. Click
          Run aggregation now in the Force-run section.
    • 
          
  • Force-run buttons return 404 — ops_routes.pb.js didn't
          load. Re-run deploy, then click Ping /api/ops/health to verify.
    • 
          
  • Crashes spike after a release — click into the Recent crashes
          list and group by error_name. Roll back the release if it's systemic.
    • 
          
  • Dashboard fetches get rate-limited — PocketHost added /api
          rate limits. Bump the refresh interval dropdown up top to 5m or off.
